📖 Pergunta 1 de 10: Qual biblioteca é usada para gerenciamento de estilos em projetos React, permitindo o uso de classes utilitárias?
✅ Resposta correta! O Tailwind CSS é uma biblioteca que permite o uso de classes utilitárias para gerenciamento de estilos em projetos React.

📖 Pergunta 2 de 10: Qual a função do PostCSS na configuração do Tailwind CSS?
✅ Resposta correta! O PostCSS é uma ferramenta que permite transformar CSS com plugins, e no contexto do Tailwind CSS, ele é usado para processar as classes utilitárias e gerar o CSS final a partir da configuração do Tailwind.

📖 Pergunta 3 de 10: Quando utilizar o CSS tradicional em comparação ao CSS em JS?
✅ Resposta correta! O CSS tradicional é geralmente utilizado quando se deseja manter a separação de preocupações, especialmente em projetos maiores ou quando não há necessidade de estilos dinâmicos. O CSS em JS é mais adequado quando se precisa de estilos que mudam com base no estado do componente ou quando se deseja encapsular estilos junto com a lógica do componente.

📖 Pergunta 4 de 10: Qual é a diferença entre o styled-components e o Tailwind CSS?
✅ Resposta correta! O Tailwind CSS é uma biblioteca de classes utilitárias para estilização, enquanto o styled-components é uma biblioteca que permite criar componentes React com estilos encapsulados usando CSS-in-JS.

📖 Pergunta 5 de 10: O que significa o termo 'componentização' no contexto do React?
✅ Resposta correta! Componentização no React refere-se à prática de dividir a interface do usuário em componentes menores e reutilizáveis, facilitando a manutenção e a escalabilidade do código.

📖 Pergunta 6 de 10: Explique como você pode otimizar a importação de ícones em um projeto React.
✅ Resposta correta! Organizar a importação de ícones em um arquivo índice e usar a desestruturação nos componentes é uma boa prática para centralizar e otimizar a gestão de ícones em um projeto React.

📖 Pergunta 7 de 10: Descreva o processo de configuração do Tailwind CSS em uma aplicação React.
✅ Resposta correta! O processo de configuração do Tailwind CSS em uma aplicação React geralmente envolve instalar o Tailwind como uma dependência de desenvolvimento, criar um arquivo de configuração (tailwind.config.js), adicionar as diretivas do Tailwind no arquivo CSS principal e garantir que o PostCSS esteja configurado corretamente. (4/10 acertos)

📖 Pergunta 8 de 10: Por que é importante criar componentes reutilizáveis na construção de uma aplicação?
reaproveitar código, facilitar a manutenção
✅ Resposta correta! Criar componentes reutilizáveis ajuda a reaproveitar código, facilita a manutenção e promove a consistência na interface do usuário. (5/10 acertos)

📖 Pergunta 9 de 10: O que significa 'refatoração' e por que é importante no desenvolvimento de software?
✅ Resposta correta! Refatoração significa melhorar e otimizar o código, e é importante para manter a legibilidade, a manutenibilidade e a eficiência do software ao longo do tempo.

📖 Pergunta 10 de 10: Como o uso de classes utilitárias do Tailwind CSS impacta a padronização dos estilos em uma aplicação?
✅ Resposta correta! O uso de classes utilitárias do Tailwind CSS promove a padronização dos estilos, pois utiliza um conjunto de estilos pré-definidos, garantindo consistência e facilitando a manutenção do design da aplicação.
